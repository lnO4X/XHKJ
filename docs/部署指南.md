# 部署指南 — 阿里云双 ECS 架构

## 服务器规划

| 服务器 | 规格 | 角色 | 运行服务 |
|--------|------|------|----------|
| **ECS-1** | 4C8G | Web 网关 + 前台 | Nginx (SSL 终止) + Nuxt 3 SSR |
| **ECS-2** | 4C8G | 后台服务 + 数据 | Strapi v5 + PostgreSQL 16 + Redis 7 |

### 为什么这样划分？

1. **职责分离**：前台面向公网高并发访问，后台面向内部管理低频操作
2. **安全隔离**：数据库和 CMS 管理端在 ECS-2，不直接暴露公网
3. **资源互不影响**：前台 SSR 渲染和数据库查询各用各的 CPU/内存
4. **独立扩容**：未来流量大了，ECS-1 可以水平扩展多台

## ECS-1 部署（Web 网关 + Nuxt 3）

### docker-compose.yml

```yaml
version: "3.8"
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - ./nginx/ssl:/etc/nginx/ssl
      - ./nginx/logs:/var/log/nginx
    depends_on:
      - nuxt
    restart: always

  nuxt:
    build:
      context: ../../xinghui-web
      dockerfile: Dockerfile
    environment:
      - NUXT_PUBLIC_STRAPI_URL=http://<ECS-2内网IP>:1337
    expose:
      - "3000"
    restart: always
```

### Nginx 配置

```nginx
# /etc/nginx/conf.d/xhmxb.conf
upstream nuxt_backend {
    server nuxt:3000;
}

upstream strapi_backend {
    server <ECS-2内网IP>:1337;
}

server {
    listen 80;
    server_name xhmxb.com www.xhmxb.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name xhmxb.com www.xhmxb.com;

    ssl_certificate     /etc/nginx/ssl/xhmxb.com.pem;
    ssl_certificate_key /etc/nginx/ssl/xhmxb.com.key;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # Nuxt SSR 前台
    location / {
        proxy_pass http://nuxt_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Strapi API（代理到 ECS-2）
    location /api/ {
        proxy_pass http://strapi_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Strapi 上传文件
    location /uploads/ {
        proxy_pass http://strapi_backend;
    }

    # Strapi Admin（可选：限制 IP 或通过 VPN 访问）
    location /admin/ {
        # allow <办公网络IP>;
        # deny all;
        proxy_pass http://strapi_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        proxy_pass http://nuxt_backend;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

## ECS-2 部署（Strapi + PostgreSQL + Redis）

### docker-compose.yml

```yaml
version: "3.8"
services:
  strapi:
    build:
      context: ../../xinghui-cms
      dockerfile: Dockerfile
    environment:
      - DATABASE_CLIENT=postgres
      - DATABASE_HOST=postgres
      - DATABASE_PORT=5432
      - DATABASE_NAME=xinghui_cms
      - DATABASE_USERNAME=xinghui
      - DATABASE_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - APP_KEYS=${APP_KEYS}
      - API_TOKEN_SALT=${API_TOKEN_SALT}
      - ADMIN_JWT_SECRET=${ADMIN_JWT_SECRET}
      - JWT_SECRET=${JWT_SECRET}
      - SMTP_HOST=${SMTP_HOST}
      - SMTP_PORT=${SMTP_PORT}
      - SMTP_USER=${SMTP_USER}
      - SMTP_PASS=${SMTP_PASS}
    volumes:
      - strapi_uploads:/opt/app/public/uploads
    ports:
      - "1337:1337"
    depends_on:
      - postgres
      - redis
    restart: always

  postgres:
    image: postgres:16-alpine
    environment:
      - POSTGRES_DB=xinghui_cms
      - POSTGRES_USER=xinghui
      - POSTGRES_PASSWORD=${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "127.0.0.1:5432:5432"  # 仅本机访问
    restart: always

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - redisdata:/data
    ports:
      - "127.0.0.1:6379:6379"  # 仅本机访问
    restart: always

volumes:
  pgdata:
  redisdata:
  strapi_uploads:
```

## 安全加固

| 措施 | ECS-1 | ECS-2 |
|------|-------|-------|
| 安全组：仅开放 80/443 | ✅ | ❌ 不开放公网 |
| 安全组：内网互通 | ✅ | ✅ |
| PostgreSQL 仅绑定 127.0.0.1 | — | ✅ |
| Redis 设密码 + 仅绑定 127.0.0.1 | — | ✅ |
| Strapi Admin 限制 IP | ✅ (Nginx) | — |
| SSL 证书（Let's Encrypt） | ✅ | — |
| 定期备份 PostgreSQL | — | ✅ crontab |

## 备份策略

```bash
# ECS-2 上的 PostgreSQL 每日备份脚本
# /opt/scripts/backup-pg.sh
#!/bin/bash
BACKUP_DIR=/opt/backups/postgres
DATE=$(date +%Y%m%d_%H%M%S)
docker exec postgres pg_dump -U xinghui xinghui_cms | gzip > $BACKUP_DIR/xinghui_cms_$DATE.sql.gz
# 保留最近 30 天
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

# crontab -e
# 0 3 * * * /opt/scripts/backup-pg.sh
```

## 部署步骤

### 首次部署

```bash
# 1. 两台 ECS 都安装 Docker + Docker Compose
curl -fsSL https://get.docker.com | sh
apt install docker-compose-plugin -y

# 2. 克隆代码
git clone https://github.com/lnO4X/XHKJ.git
cd XHKJ

# 3. ECS-2 先启动数据服务
cd deploy/ecs-2
cp .env.example .env  # 编辑填入密码
docker compose up -d

# 4. ECS-1 启动 Web 服务
cd deploy/ecs-1
cp .env.example .env  # 填入 ECS-2 内网 IP
docker compose up -d

# 5. 配置 SSL 证书
# 方案一：阿里云免费证书下载放到 nginx/ssl/
# 方案二：certbot 自动签发
```

### 更新部署

```bash
# 拉取最新代码
git pull origin main

# 重新构建并启动（零停机）
docker compose up -d --build
```
