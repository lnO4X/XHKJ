# 设计文档 — 薪汇科技官网

> 描述"做什么"和"为什么"。需求来源、产品范围、用户流程、UI 线框。
> 技术实现细节 → 见 [TECH.md](./TECH.md)

---

## 一、项目背景

- **公司**：深圳市民信惠集团旗下薪汇科技
- **现状**：xhmxb.com 是一个 HR SaaS（民薪保）网站，不支持响应式
- **目标**：改版为"数据服务商"定位的科技官网
- **参考**：shuxin-tech.com（深圳数鑫科技 — 数据空间 / 数据可信流通）

### 需求来源（微信对话摘要）

| 角色 | 原话关键信息 |
|------|-------------|
| 业务方 | "我想把他改成数商的网站"、"薪汇的公众号也改" |
| 业务方 | "这两个网站响应式都不支持" |
| 技术方 | "邮箱限制民信惠域名，验证码登录是最简单的" |
| 技术方 | "我考虑还是尽可能的简单" |
| 技术方 | "你还有一个后台编辑文章" |
| 决策 | "使用旧的域名"、"咱们自己搞" |

## 二、产品范围

### 2.1 MVP 范围（P0）

| 功能 | 说明 |
|------|------|
| 官网前台 5 页 | 首页、产品服务、新闻动态、关于我们、联系我们 |
| 响应式布局 | PC / 平板 / 手机三端自适应 |
| 后台 CMS | 文章发布、Banner 管理、产品管理、公司信息编辑 |
| 管理员登录 | 邮箱验证码登录（限 @minxinhui.com） |
| 微信服务号 | 菜单链接改版（P1 可延后） |

### 2.2 不做（Out of MVP）

| 排除项 | 原因 |
|--------|------|
| 前台用户注册/登录 | 展示型官网不需要 |
| 多语言 | 首期仅中文 |
| RBAC 多角色权限 | 首期仅管理员单角色 |
| 公众号图文自动同步 | 首期手动操作 |
| 在线交易/支付 | 纯展示站 |

### 4.4 响应式断点

| 断点 | 宽度 | 布局 | 导航 |
|------|------|------|------|
| 手机 | ≤ 767px | 单栏堆叠 | 汉堡菜单 |
| 平板 | 768-1199px | 双栏网格 | 精简菜单 |
| PC | ≥ 1200px | 全宽多栏 | 完整水平导航 |

### 4.5 视觉风格

- **主色**：科技蓝 #0052D9
- **辅色**：深空灰 #1A1A2E
- **背景**：白色 #FFFFFF / 浅灰 #F7F8FA
- **文字**：主文字 #1A1A2E / 辅助文字 #8590A6
- **字体**：系统字体栈（PingFang SC, -apple-system, sans-serif）
- **动效**：滚动渐入、数字滚动、卡片悬浮

### M-WEB 适配要点

| 要点 | 措施 |
|------|------|
| viewport | `width=device-width, initial-scale=1, maximum-scale=5` |
| 触摸区域 | 按钮最小 44px |
| 字体大小 | 正文 ≥ 14px，标题 ≥ 18px |
| 底部安全区 | `env(safe-area-inset-bottom)` |
| 微信分享 | OpenGraph meta + JSSDK 自定义分享 |
| 图片加载 | 懒加载 + WebP 格式 |

## 六、管理后台流程

### 6.1 登录流程

```
管理员 → 输入 @minxinhui.com 邮箱
       → 系统发送 6 位验证码 (5min 有效)
       → 输入验证码
       → 签发 JWT → 进入后台
```

### 6.2 文章发布流程

```
新建 → 编辑标题/分类/正文/封面/摘要
     → 保存草稿
     → 预览
     → 发布（设置 publishedAt）
     → 可下架/重新发布
```

### 6.3 后台功能清单

| 模块 | 操作 |
|------|------|
| Banner 管理 | 增删改查、拖拽排序、上下架 |
| 产品管理 | 增删改查、图标/封面上传、排序 |
| 文章管理 | 增删改查、富文本编辑、分类筛选、置顶、草稿/发布 |
| 分类管理 | 增删改查 |
| 公司信息 | 编辑单例（名称、简介、地址、电话等） |
| 首页配置 | 编辑单例（Hero 标题、数字亮点） |
| 留言管理 | 列表查看、标记已读 |
