## 文档规范

### 格式约定

每个文档顶部必须包含 YAML front-matter 元数据块：

```yaml
---
id: ENV                       # 唯一标识，用于文档间引用
title: 环境信息
version: 1.0.0                # 语义化版本
updated: 2026-03-05           # 最后更新日期
status: draft | review | approved
owner: Eason                  # 维护人
depends_on: []                # 依赖的其他文档
---
```

### 交叉引用

文档间使用 `→ 见 [TECH.md#2.3]` 格式引用，避免内容重复。

### 待确认项

使用统一标记 `[TODO]` 表示待确认/待填入的信息，便于 agent 搜索：

```
服务器公网 IP: [TODO - 待运维提供]
```

### 敏感信息

密码、Token、密钥等敏感信息 **不得** 写入文档。使用占位符：

```
SMTP_PASS=[CREDENTIAL:minxinhui-smtp-password]
```

实际值存放在各环境的 `.env` 文件中（不纳入 Git）。
